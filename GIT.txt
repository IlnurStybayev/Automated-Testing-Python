1. git config
   git config --global user.name "Ваше Имя"
   git config --global user.email "ваш_email@example.com"
   Устанавливает глобальные настройки пользователя для Git.

2. git init
   Инициализирует новый репозиторий в текущем каталоге.

3. git clone
   Клонирует удаленный репозиторий на локальную машину.

4. git add
   Добавляет файл в индекс для последующего коммита.

5. git commit
   Создает коммит с изменениями в индексе. Флаг `-m` используется для добавления сообщения к коммиту.

6. git status
   Показывает состояние рабочего каталога и индекса.

7. git log
   Выводит историю коммитов.

8. git diff
   Показывает изменения между рабочим каталогом и последним коммитом.

9. git branch
   Показывает список локальных веток.

10. git checkout
    Переключается на указанную ветку.

11. git merge
    Объединяет указанную ветку с текущей.

12. git pull
    Вытягивает изменения из удаленного репозитория и объединяет их с текущей веткой.

13. git push
    Отправляет локальные коммиты в удаленный репозиторий.

14. git remote -v
    Показывает список удаленных репозиториев и их URL.

15. git reset
    Сбрасывает индексированные изменения в указанных файлах.

16. git stash
    Сохраняет текущие изменения в стэше, позволяя переключиться на другую ветку без коммита.

17. git show коммит
    Показывает информацию о конкретном коммите.

18. git fetch
    Загружает изменения из удаленного репозитория, но не объединяет их с текущей веткой.

19. git branch -d ветка
    Удаляет локальную ветку. Флаг `-d` обозначает безопасное удаление, который не удалит ветку, содержащую неприменённые изменения.

20. git push --force origin ветка
    Принудительно отправляет локальные изменения в удаленный репозиторий, перезаписывая историю. Используйте осторожно, так как это может привести к потере данных.

21. git log --graph --oneline --all
    Выводит историю коммитов в виде графа, что полезно для визуализации структуры ветвлений.

22. git blame файл
    Показывает, кто и когда внес изменения в указанный файл.

23. git tag -a версия -m "Описание"
    Создаёт аннотированный тег для определённого коммита. Теги обычно используются для маркировки релизов.

24. git log --grep "строка"
    Фильтрует историю коммитов по указанной строке в коммит-сообщениях.

25. git remote add название_удаленного_репо URL
    Добавляет удаленный репозиторий с указанным названием и URL.
